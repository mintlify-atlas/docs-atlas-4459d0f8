This example builds a complete three-screen application from scratch using Compose Svelted's navigation system. You will define typed routes, create individual screen components that receive a `NavController`, pass arguments between screens, and wire everything together inside a `NavHost` with animated transitions. By the end you will have a working Login → Home → Details flow with full back-stack support.

<Note>
  The navigation API mirrors Jetpack Compose's `NavController` / `NavHost` pattern. If you have used `androidx.navigation` in Android development, the concepts will feel immediately familiar.
</Note>

<Steps>
  <Step title="Define your routes">
    Start by declaring your route objects in a shared `routes.ts` file. Each route is a plain object with a `path` string that satisfies the `Route` type contract. Exporting them as individual named constants lets every screen import from one source of truth and avoids hard-coded path strings.

    ```ts
    // routes.ts
    import type { Route } from "@danielito1996/compose-svelted";

    export const Login:   Route = { path: "login" };
    export const Home:    Route = { path: "home" };
    export const Details: Route = { path: "details" };
    ```

    <Tip>
      Importing the `Route` type from the package and annotating each constant ensures TypeScript will catch any missing `path` property at compile time.
    </Tip>
  </Step>

  <Step title="Create LoginScreen">
    The login screen collects credentials and navigates to `Home` on success, passing the entered email as a route argument so the next screen can greet the user by name.

    ```svelte
    <!-- LoginScreen.svelte -->
    <script lang="ts">
      import {
        Column, Text, Button, TextField, Modifier, Alignment, Arrangement
      } from "@danielito1996/compose-svelted";
      import type { NavController } from "@danielito1996/compose-svelted";
      import { Home } from "./routes";

      export let navController: NavController;
      let email = "";
      let password = "";
    </script>

    <Column
      modifier={Modifier.fillMaxSize().padding(32)}
      verticalArrangement={Arrangement.spacedBy(20)}
      horizontalAlignment={Alignment.CenterHorizontally}
    >
      <Text textStyle="headlineMedium">Sign In</Text>

      <TextField
        value={email}
        onValueChange={(v) => email = v}
        label="Email"
        modifier={Modifier.fillMaxWidth()}
      />

      <TextField
        value={password}
        onValueChange={(v) => password = v}
        label="Password"
        modifier={Modifier.fillMaxWidth()}
      />

      <Button
        onClick={() => navController.navigate(Home.path, { user: email })}
        modifier={Modifier.fillMaxWidth()}
      >
        <Text>Sign In</Text>
      </Button>
    </Column>
    ```

    The second argument to `navController.navigate()` is a free-form object. The receiving screen reads it back from `navBackStackEntry.args` — covered in the next step.
  </Step>

  <Step title="Create HomeScreen">
    `HomeScreen` receives the `navBackStackEntry` prop automatically from `NavHost`. It reads `user` from `args` to display a personalised greeting, then offers navigation to `Details` (passing an item ID) or a logout action that pops the back stack.

    ```svelte
    <!-- HomeScreen.svelte -->
    <script lang="ts">
      import {
        Column, Text, Button, Modifier, Arrangement
      } from "@danielito1996/compose-svelted";
      import type { NavController } from "@danielito1996/compose-svelted";
      import { Details, Login } from "./routes";

      export let navController: NavController;
      export let navBackStackEntry: { route: string; args?: { user: string } };

      const user = (navBackStackEntry?.args ?? {}).user ?? "";
    </script>

    <Column
      modifier={Modifier.fillMaxSize().padding(24)}
      verticalArrangement={Arrangement.spacedBy(16)}
    >
      <Text textStyle="headlineMedium">Welcome, {user}!</Text>

      <Button onClick={() => navController.navigate(Details.path, { id: 1 })}>
        <Text>View Details</Text>
      </Button>

      <Button onClick={() => navController.navigate(Login.path)}>
        <Text>Log Out</Text>
      </Button>
    </Column>
    ```

    <Warning>
      Always guard `navBackStackEntry.args` with optional chaining before accessing properties — `navBackStackEntry?.args ?? {}` — because args is `undefined` when a screen is the initial destination and was not navigated to with arguments.
    </Warning>
  </Step>

  <Step title="Create DetailsScreen">
    `DetailsScreen` reads the `id` argument and renders item-specific content. A single back button calls `popBackStack()` to return the user to `HomeScreen`.

    ```svelte
    <!-- DetailsScreen.svelte -->
    <script lang="ts">
      import {
        Column, Text, Button, Modifier, Arrangement
      } from "@danielito1996/compose-svelted";
      import type { NavController } from "@danielito1996/compose-svelted";

      export let navController: NavController;
      export let navBackStackEntry: { route: string; args?: { id: number } };

      const { id } = navBackStackEntry?.args ?? {};
    </script>

    <Column
      modifier={Modifier.fillMaxSize().padding(24)}
      verticalArrangement={Arrangement.spacedBy(16)}
    >
      <Text textStyle="headlineMedium">Details: item #{id}</Text>

      <Button onClick={() => navController.popBackStack()}>
        <Text>Go Back</Text>
      </Button>
    </Column>
    ```
  </Step>

  <Step title="Wire everything together in App.svelte">
    Finally, instantiate a `NavController` with `rememberNavController`, declare the route-to-component map using `composable`, and hand both to `NavHost`. `ComposeTheme` and `AppRoot` must wrap the entire tree.

    ```svelte
    <!-- App.svelte -->
    <script lang="ts">
      import {
        ComposeTheme, AppRoot, NavHost,
        rememberNavController, composable, slideHorizontal
      } from "@danielito1996/compose-svelted";
      import LoginScreen from "./LoginScreen.svelte";
      import HomeScreen from "./HomeScreen.svelte";
      import DetailsScreen from "./DetailsScreen.svelte";
      import { Login, Home, Details } from "./routes";

      const navController = rememberNavController(Login.path);

      const routes = [
        composable(Login,   () => LoginScreen),
        composable(Home,    () => HomeScreen),
        composable(Details, () => DetailsScreen),
      ];
    </script>

    <ComposeTheme mode="system">
      <AppRoot>
        <NavHost
          {navController}
          {routes}
          transition={slideHorizontal()}
        />
      </AppRoot>
    </ComposeTheme>
    ```

    The `transition` prop accepts any built-in transition factory — `slideHorizontal()`, `scaleFade()`, or `fade()` — or a custom `ContentTransition` object. Setting it here applies the same animation to every route change in this host.

    <Tip>
      `rememberNavController` takes the **initial route path** as its argument. The controller will navigate there automatically on first render, so make sure the path matches one of your `composable` entries exactly.
    </Tip>
  </Step>
</Steps>

## Back-stack behaviour

`popBackStack()` removes the current screen and reveals the previous one. However, **it will not remove the last remaining screen** — if the back stack has only one entry, calling `popBackStack()` is a no-op, preventing a blank host. Design your logout or unauthenticated flows to navigate *to* a destination (e.g. `navController.navigate(Login.path)`) rather than relying on popping back to it.

## Full navigation flow diagram

```
[LoginScreen]
     │  navigate(Home.path, { user: email })
     ▼
[HomeScreen]
     │  navigate(Details.path, { id: 1 })
     ▼                                          popBackStack()
[DetailsScreen] ─────────────────────────────▶ [HomeScreen]
                                                     │ navigate(Login.path)
                                                     ▼
                                               [LoginScreen]
```
